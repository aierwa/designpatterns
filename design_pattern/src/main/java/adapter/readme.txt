
适配器模式

howtodo：如何实现，原理

场景1：封装有设计缺陷的接口服务
    - 有静态函数
    - 命名不好
    - 参数过多
    - 方法性能不好

场景2：统一多个类的接口设计
    - 多个类是做同一类事情，但是方法命名、传参等不同
    - 这里引入多个第三方敏感词过滤器，需要将其统一接口

场景3：替换依赖的外部系统
    - 外部服务有接口 IA 和实现类 A
    - 现在要把这个实现类替换了，换成另一个服务 B，方法调用全都不变
    - 可以设计一个 BAdaptor 实现 IA 接口，再组合 B

场景4：兼容老版本接口
    - 老版本的接口 去调用 新版本的接口
